🔐احراز هویت

Token Storage Strategy — Cookie Based

طبق PDFهای بک‌اند، Login توکن‌ها را داخل JSON Response برمی‌گرداند و بک‌اند خودش Set-Cookie انجام نمی‌دهد. بنابراین فرانت بعد از Login باید access_token و refresh_token را داخل Cookie ذخیره کند. در این پروژه توکن‌ها نباید داخل localStorage یا sessionStorage ذخیره شوند. 

// login success
Cookies.set("access_token", data.access);
Cookies.set("refresh_token", data.refresh);

// api.ts
const token = Cookies.get("access_token");
if (token) {
  config.headers.Authorization = `Bearer ${token}`;
}
قانون مهم Auth

Do not use localStorage for tokens. Token storage must be Cookie-based.

✅Backend Confirmed APIs

ابهام‌های قبلی بک‌اند برای MVP حل شده‌اند و موارد زیر تأیید شده‌اند:

1. API دریافت صندلی‌های سفر

برای صفحه انتخاب صندلی، بک‌اند این endpoint را تأیید کرده است:

GET/api/trips/company/trips/{id}/seats/
[
  {
    "id": 1,
    "number": 1,
    "status": "available"
  },
  {
    "id": 2,
    "number": 2,
    "status": "reserved"
  }
]
2. منطق قفل شدن صندلی در pending_payment

بک‌اند تأیید کرده است که هنگام ساخت Booking، عملیات به شکل atomic انجام می‌شود. اگر User A صندلی ۵ را انتخاب کند و Booking با وضعیت pending_payment ساخته شود، همان صندلی برای همان Trip قفل می‌شود. 

User A
Seat 5
Booking Status = pending_payment

↓

Seat 5 is locked

↓

User B cannot book Seat 5 for the same Trip

در نتیجه رزرو تکراری یک صندلی برای یک سفر امکان‌پذیر نیست.

3. API دریافت اطلاعات کاربر لاگین‌شده

برای Session Restore، Protected Routes و Role Guard، بک‌اند endpoint زیر را تأیید کرده است:

GET/api/accounts/me/
Authorization: Bearer <jwt_token>

Response — Regular User

{
  "id": 5,
  "first_name": "Ali",
  "last_name": "Ahmadi",
  "phone": "09123456789",
  "email": "ali@example.com",
  "role": "user",
  "company_name": null
}

Response — Company Owner

{
  "id": 3,
  "first_name": "Mohammad",
  "last_name": "Rezaei",
  "phone": "09987654321",
  "email": "mohammad@travelco.com",
  "role": "owner",
  "company_name": "Pars Travel Co."
}
4. نام شهرها در خروجی Trip

بک‌اند تأیید کرده است که خروجی Trip هم ID شهر و هم نام شهر را برمی‌گرداند:

{
  "id": 12,
  "origin_city": 1,
  "origin_city_name": "Tehran",
  "destination_city": 2,
  "destination_city_name": "Shiraz"
}

در کارت سفر و صفحه جزئیات سفر باید از origin_city_name و destination_city_name برای نمایش استفاده شود.

5. Search by City Name و نبود API شهرها

برای MVP، کاربر نام شهر را داخل فرم تایپ می‌کند. بنابراین در فرم جستجو نیازی به API لیست شهرها نیست.

GET/api/trips/company/trips/search/?origin=Tehran&destination=Shiraz&date=2026-06-15

Autocomplete یا API شهرها می‌تواند بعداً به عنوان بهبود UX اضافه شود، اما برای MVP بلاکر نیست.

services/api.ts — Axios Instance

import axios from "axios";
import Cookies from "js-cookie";

export const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_BASE_URL,
});

api.interceptors.request.use((config) => {
  const token = typeof window !== "undefined"
    ? Cookies.get("access_token") : null;

  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

🐻Store — Zustand

store/auth.store.ts

interface AuthState {
  user: Account | null;
  accessToken: string | null;
  isAuthenticated: boolean;
  setAuth: (data: { user, accessToken, refreshToken }) => void;
  logout: () => void;
}

export const useAuthStore = create<AuthState>((set) => ({
  // ذخیره tokens در Cookies + state
  setAuth: ({ user, accessToken, refreshToken }) => {
    Cookies.set("access_token", accessToken);
    Cookies.set("refresh_token", refreshToken);
    set({ user, accessToken, isAuthenticated: true });
  },
  logout: () => {
    Cookies.remove("access_token");
    Cookies.remove("refresh_token");
    set({ user: null, accessToken: null, isAuthenticated: false });
  },
}));

🪝Hooks & Patterns

نمونه Query Hook

import { useQuery } from "@tanstack/react-query";
import { tripsService } from "@/services/trips.service";
import { QUERY_KEYS } from "@/constants/queryKeys";

export function useTripDetails(id: string | number) {
  return useQuery({
    queryKey: QUERY_KEYS.trip(id),
    queryFn: () => tripsService.getById(id),
    enabled: Boolean(id),
  });
}

نمونه Mutation Hook

import { useMutation, useQueryClient } from "@tanstack/react-query";

export function useCreateBooking() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: bookingService.create,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: QUERY_KEYS.bookings });
    },
  });
}

app/providers.tsx — QueryClient + Toaster

"use client";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { Toaster } from "sonner";

export function Providers({ children }) {
  const [queryClient] = useState(() => new QueryClient());
  return (
    <QueryClientProvider client={queryClient}>
      {children}
      <Toaster richColors position="top-center" />
    </QueryClientProvider>
  );
}

// app/layout.tsx — RTL
<html lang="fa" dir="rtl">
  <body><Providers>{children}</Providers></body>
</html>
